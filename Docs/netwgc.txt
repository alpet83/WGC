

   Программа делиться на двухкомпонентную систему окончательно.
Ядро основанное на библиотеке chdip.dll теперь представляет собой
серверную часть, а оболочка wgc.exe клиентскую. Связь их может осуществлятся
с помощью сокетного соединения (в перспективе будет реализован механизм передачи
сообщений и данных посредством MMF).


 primary tasks:
 1. Оптимизация обмена данными для таблицы значений (watch/cheat table).
 2. Реализация управления сервером:
  Нужно написать интерфейс интерактивного взаимодействия с пользователем, посредством
 консоли. Обработка ввода должна осуществляться в основном потоке.
